## STOP-ON-BLOCKER RULE

If any blocker happens, stop immediately.

A blocker includes:

* Route cannot load
* API returns 4xx/5xx and page cannot continue
* Required public data does not appear
* Booking cannot continue
* UI says success but backend does not confirm
* Mock/fake/timer success is used
* Public route exposes private/sensitive data

1. Stop.
2. Do not fix.

5. End with `STOPPED_ON_BLOCKER`.

### Public booking

3. Select real service.

5. Select valid slot.

2. Refresh the page.
3. Confirm confirmation still loads.
4. Confirm it uses safe public endpoint.
5. Confirm it does not expose full patient profile, medical records, private notes, or sensitive payment details.
6. Confirm back/home/account buttons work.

### Privacy policy

1. Open `/public/privacy-policy`.
2. Confirm page loads.
3. Confirm back/home links work.

### Login/Register/Forgot/Reset

1. Open `/auth/login`.
2. Confirm login form appears.
3. Confirm forgot password link works.
4. Open `/auth/forgot-password`.
5. Submit a test email.
6. Confirm real backend API is called.
7. Confirm safe success message does not reveal if email exists.
8. Open `/auth/reset-password`.
9. If no valid token exists, confirm validation/error behavior is correct.
10. Confirm it does not fake success.
11. Open `/auth/register`.
12. Confirm register form loads.
13. Do not create duplicate patient unless safe test data is used.

### Unauthorized access checks

As Public/Guest, attempt:

* `/admin/dashboard`
* `/staff/dashboard`
* `/doctor/dashboard`
* `/patient/dashboard`
