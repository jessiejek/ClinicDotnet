## 2. Root Cause

The booking flow had two parallel code paths that were partially migrated.

**Path A — step-payment.component.ts (PayAtClinic flow)** ✅ FIXED
- Old behavior: sent empty `{}` POST to `bookings`, read `booking_id` from response
- Old API call: `apiService.post<any>('bookings', {})`
- Old payload: `{}` (empty)
- Old response parsing: `createdRow?.['booking_id']`
- Why it failed: .NET backend needs real payload fields (doctorId, serviceIds, appointmentDate, slotStartTime, slotEndTime) and returns `id` not `booking_id`
- Fix: Sends proper payload with all required fields, reads `booking?.id` from response

**Path B — step-proof.component.ts (Online/Proof payment flow)** ❌ STILL BROKEN
- Same file pattern but was NOT updated alongside step-payment
- Still calls `apiService.post<any>('bookings', {})` with empty payload
- Still reads `createdRow?.booking_id`
- Will fail with .NET backend

**Path C — admin/walk-in/walk-in.page.ts** ❌ STILL BROKEN
- Same empty `{}` POST pattern
- Same `booking_id` response parsing
- Will fail with .NET backend

**Path D — staff/walk-in/staff-walk-in.page.ts** ❌ STILL BROKEN
- Same empty `{}` POST pattern
- Same `booking_id` response parsing
- Will fail with .NET backend

## 4. API Contract Used (for Path A — step-payment only)

Endpoint: POST /api/bookings
Method: POST
Frontend caller: `step-payment.component.ts` → `submitBookingRequest()`
Request payload:
```json
{
  "doctorId": "guid",
  "serviceIds": ["guid", ...],
  "appointmentDate": "2026-05-28",
  "slotStartTime": "09:00",
  "slotEndTime": "09:30",
  "notes": "string|null"
}
```
Response shape:
```json
{
  "id": "guid",
  "patientId": "guid",
  "doctorId": "guid",
  ...
}
```
Booking ID source: `booking?.id` (camelCase .NET convention)
Status: CORRECT for step-payment flow
